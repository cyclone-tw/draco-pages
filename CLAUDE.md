# draco-pages

Draco 產出的一頁式說明網頁，透過 GitHub Pages 部署。

## 用途

OpenClaw 的 Draco agent 透過 Claude Code 產出一頁式 HTML 網頁，push 到此 repo 後自動部署。

## 網頁規範

### 檔案結構

```
draco-pages/
├── index.html          ← 首頁（頁面索引）
├── pages/
│   └── <slug>/
│       └── index.html  ← 每個一頁式網頁
└── CLAUDE.md
```

### 每個頁面的要求

1. **自包含 HTML** — 所有 CSS 用 `<style>` inline，不引用外部 CSS/JS 檔案
2. **唯一例外** — 可引用 Google Fonts（Noto Sans TC + Inter）
3. **響應式設計** — 手機、平板、桌機都能看
4. **深淺色自適應** — 使用 `prefers-color-scheme` media query
5. **繁體中文** — 主要語言，技術名詞保留英文
6. **風格** — 現代簡潔、大量留白、圓角卡片、漸層 accent color

### 常見區塊結構

- **Hero** — 大標題 + 副標題 + 簡短描述
- **Features / 重點** — icon/emoji + 標題 + 說明的卡片網格
- **步驟說明** — 編號步驟 + 視覺化流程
- **內容主體** — 文章內容段落
- **CTA / 結尾** — 行動呼籲或結語
- **Footer** — 「Generated by Draco 🐉 | Powered by OpenClaw」

### index.html（首頁）

首頁是所有頁面的索引，每次新增頁面後必須更新。格式：

```html
<!-- 每個頁面一個卡片連結 -->
<a href="pages/<slug>/">
  <h3>標題</h3>
  <p>簡短描述</p>
  <span>日期</span>
</a>
```

## 部署

- GitHub Pages 從 `main` branch 根目錄部署
- URL 格式：`cyclone-tw.github.io/draco-pages/pages/<slug>/`
- Push 到 main 後約 1-2 分鐘自動部署

## 工作流程

1. 收到 Draco 的規格 prompt（主題、內容重點、風格、受眾）
2. 在 `pages/<slug>/index.html` 產出自包含 HTML
3. 更新 `index.html` 首頁索引
4. Commit + push
5. 回報 GitHub Pages URL
